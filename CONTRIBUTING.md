# Contributing to Capacitarr

Thank you for your interest in contributing to Capacitarr!

## Contributor License Agreement (CLA)

Before we can merge any of your contributions (pull requests, patches, etc.), you must sign our Contributor License Agreement (CLA). This ensures that we have the necessary rights to use and distribute your contributions under our project's license, while you retain ownership of your original work.

By submitting a pull request, you confirm that you have read and agree to the terms of the CLA. If you have not signed the CLA, the automated checks on your pull request will fail, and we will provide instructions on how to sign it.

### General Contribution Guidelines

1. **Issues First**: Please open an issue to discuss proposed changes before starting work on a major feature.
2. **Code Style**: Please adhere to the existing code formatting conventions. Use the established style for Go and Vue/Nuxt.
3. **Tests**: Ensure that any new functionality is accompanied by appropriate tests. Run the test suite before submitting a pull request.
4. **Commits**: Write clear, descriptive commit messages.

We appreciate all contributions, whether they are bug fixes, new features, or documentation improvements!
